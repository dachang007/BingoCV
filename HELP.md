**现在程序需要哪些训练特征**
目前训练代码要求这些列：

```
GR平均值 GR_Lh平均值 GR范围 AC_Lh平均值 DEN平均值 DEN_Lh平均值 DEN范围 分类
```

这里 分类 必须是：

```
主体 侧翼
```

如果你原始井有很多参数，比如 20 个、30 个，不影响识别。当前随机森林只用上面这 7 个特征，多余参数暂时不会用。

**一口井够不够训练**
一般不够。

一口井的数据容易让模型只学到这一口井的特征习惯，换井后泛化差。建议：

- 最低体验：几十个标注层段，主体/侧翼都要有。
- 初步可用：5-10 口井，至少 100-300 个标注层段。
- 比较稳：20 口井以上，几百到上千个标注层段。
- 主体和侧翼样本数量不要严重失衡。

如果一口井只有十几个层段，训练出来的准确率可能看着高，但实际换井会不稳定。

**训练用 10 个参数，识别井有 20 个参数可以吗**
可以，但前提是识别井里必须包含模型需要的关键列，比如：

```
Depth GR SH
```

最好还有：

```
AC DEN
```

多出来的 20 个参数不会影响，程序会忽略不用的列。

反过来不行：如果训练/识别需要 GR、AC、DEN、SH，但某口井缺 DEN 或 AC，模型特征会不完整，只能 fallback 到规则或出现精度下降

现在 数据处理 页新增了 识别参数 区域，包含：

- SH阈值：小于该值按砂岩候选
- 齿化窗口：计算齿化率时前后取多少个深度点
- 薄层点数：短砂层过滤阈值
- 主体最小厚度：主体过薄时转为侧翼
- 厚层分割阈值：达到该厚度后尝试自动分割
- 规则GR_Lh阈值：模型不可用或特征缺失时的兜底规则
- 薄层过滤：是否启用短砂层过滤
- 厚层分割：是否启用厚层自动分割
- 允许规则兜底：模型/特征不完整时是否允许用规则判断
